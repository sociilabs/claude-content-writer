# Twitter/X Content Conventions

Guidelines for creating tweets and Twitter threads that drive engagement and sound authentically human.

---

## Single Tweet

### Format defaults
- **Length:** 240-280 characters for maximum engagement. Twitter's algorithm favors tweets that get quoted/retweeted, and shorter tweets are easier to quote.
- **No hashtags in body:** Place 1-2 hashtags in a reply to your own tweet instead. Hashtags in the main tweet reduce engagement.
- **Hook-first:** Lead with the most interesting part. No setup, no context — straight to the insight.
- **Line breaks:** Use 1-2 strategic line breaks for readability and emphasis.

### Structure patterns

**Pattern 1: Insight + proof point**
```
Most founders waste $40K on infrastructure they'll rebuild in 6 months.

The fix: Start with managed services. Scale to custom when revenue justifies it.
```

**Pattern 2: Contrarian take**
```
Hot take: Your MVP doesn't need real-time features.

Async + polling works fine until 10K users. Don't optimize for scale you don't have yet.
```

**Pattern 3: Specific scenario**
```
Client came to us after spending 8 months building a platform.

Zero users. Zero revenue. Beautiful code nobody wanted.

This is why we start with a landing page and 50 customer conversations.
```

**Pattern 4: Framework tease**
```
We use a 3-question framework before writing any code:

1. Who will pay for this?
2. What's the smallest version they'd pay for?
3. Can we build it in 6 weeks?

Saves months of wasted development.
```

### Voice guidelines
- **Be direct:** No filler words. Every word earns its place.
- **Use numbers:** Specific metrics > vague claims. "$40K" beats "expensive."
- **Avoid AI tells:** No "delve," "landscape," "crucial," "leverage," "robust."
- **First person when relevant:** "We see this constantly" > "This is commonly observed."
- **Conversational:** Write like you're explaining to a colleague, not presenting at a conference.

### Engagement tactics
- **Ask questions sparingly:** Only if you genuinely want replies. Empty "What do you think?" questions feel engagement-baity.
- **Tag people only when relevant:** Don't tag for visibility. Tag when their work is directly referenced.
- **Quote tweet with value:** If QT'ing someone, add a specific insight or example — not just "This!" or "Great point."

## Twitter Thread

### Format defaults
- **Length:** 5-12 tweets. Shorter threads (5-7) perform better. Only go longer if the content demands it.
- **First tweet is standalone:** It should work as a single tweet. Many people won't click through.
- **Numbered tweets:** "1/7" format in each tweet. Helps readers track progress.
- **Hook in tweet 1:** The thread lives or dies on the first tweet. Make it count.
- **No "Thread 🧵" announcement:** Just start the thread. The content should make it obvious.

### Structure

**Tweet 1: The hook**
- Surprising stat, bold claim, or specific scenario
- Should make someone think "I need to read the rest"
- Include the thread count: "1/7"

**Tweets 2-3: The problem/context**
- Why this matters
- What most people get wrong
- Specific example or case study reference

**Tweets 4-8: The framework/solution**
- Numbered steps or key insights
- One main idea per tweet
- Specific metrics and examples
- Use → arrows for sub-points within a tweet

**Tweet 9-10: The proof**
- Case study result
- Specific outcome with numbers
- "We did X, got Y" format

**Final tweet: The takeaway + CTA**
- Reframe the stakes
- Soft CTA if appropriate (link to blog, booking, etc.)
- "End of thread. [Optional CTA]"

### Thread example structure

```
Tweet 1/7:
Most SaaS founders waste 6 months building features nobody wants.

Here's the framework we use to validate ideas in 3 weeks instead:

Tweet 2/7:
The problem: Founders confuse "people said they'd use it" with validation.

Validation = money or time invested. Everything else is politeness.

Tweet 3/7:
Our 3-week validation sprint:

Week 1: 20 customer interviews
Week 2: Landing page + waitlist
Week 3: Paid pilot with 3-5 customers

Tweet 4/7:
Week 1 — Customer interviews

Goal: Find the expensive problem, not validate your solution.

Ask:
→ What's your current process?
→ What breaks?
→ What does that cost you?

Tweet 5/7:
Week 2 — Landing page + waitlist

Build a page that describes the solution. Drive 100-200 visitors.

Target: 20%+ email capture rate. If you can't get that, the problem isn't painful enough.

Tweet 6/7:
Week 3 — Paid pilot

Offer to build a basic version for 3-5 customers. Charge something (even $500).

If nobody will pay, you don't have a business yet.

Tweet 7/7:
Real example: Client validated a compliance automation tool this way.

→ 18 interviews
→ 34% waitlist conversion
→ 4 paid pilots at $2K each

Saved 4 months of speculative development.

End of thread.

Full case study: [link]
```

### Publishing notes

**Timing:** Tuesday-Thursday, 9 AM - 12 PM in target audience timezone. Avoid weekends and Mondays.

**Thread publishing:**
- Write all tweets in advance
- Post tweet 1
- Reply to tweet 1 with tweet 2
- Reply to tweet 2 with tweet 3, etc.
- Do this quickly (within 2-3 minutes) so the thread appears complete

**Engagement in first hour:**
- Reply to every comment
- Retweet the first tweet after 2-3 hours to surface it again
- Pin the thread to your profile if it's performing well

**Hashtags:** Add 1-2 relevant hashtags in a reply to the final tweet, not in the thread itself.

## Voice calibration for Twitter

Twitter has a distinct voice compared to blog articles:

- **Shorter sentences:** Twitter rewards punchy, declarative statements.
- **More direct:** Less setup, less context. Get to the point.
- **More conversational:** Contractions, fragments, casual tone.
- **More opinionated:** Twitter rewards strong takes. "I think X" > "X could potentially be considered."

When adapting blog content to Twitter:
1. Extract the most provocative claim
2. Strip all qualifiers and hedging
3. Add a specific number or example
4. Cut everything else

---

## What to avoid

**AI tells on Twitter:**
- Em dashes (—) — Use periods or line breaks instead
- "Delve," "landscape," "crucial," "robust," "seamless"
- Triple anaphora ("We need... We need... We need...")
- Emoji decoration (🚀 **Key Point:**)
- "Here's the thing:" / "Let me be clear:" throat-clearing
- Inspirational vagueness ("The future is bright!")

**Engagement-bait patterns:**
- "Retweet if you agree"
- "Like this if..."
- Empty questions ("What do you think?")
- "This will get me canceled but..."
- Artificial controversy for clicks

**Over-threading:**
- Don't turn a single tweet into a 3-tweet thread
- Don't thread just to thread
- If it fits in one tweet, keep it as one tweet

## Shortcode handling

If the user has defined shortcodes in their profile:
- `{{booking_link}}` → Replace with actual URL
- `{{case_study:name}}` → Replace with case study URL
- `{{cta:soft}}` → Adapt to Twitter's character limit and voice

Twitter CTAs should be softer and more natural than blog CTAs. "DM me if you're dealing with this" > "Book a call at [link]."

---

## Research and inspiration mode

When asked to create tweets from a URL or article:

1. **Extract the core insight:** What's the single most interesting/surprising/useful point?
2. **Find the proof:** What specific number, example, or case study supports it?
3. **Reframe for Twitter:** Make it more direct, more opinionated, more conversational.
4. **Check for AI tells:** Run the anti-AI checklist even on short-form content.

Don't just summarize the article. Find the tweet-worthy insight and build around that.
